# 任务管理系统学习文档

## 1. 编程基础知识

### 1.1 什么是编程？
编程就像是给计算机写指令，告诉它要做什么。就像我们用中文和别人交流一样，我们需要用计算机能理解的语言来和计算机交流。这些语言就叫做编程语言，比如Java。

### 1.2 Java编程语言基础

#### 1.2.1 Java是什么？
Java是一种广泛使用的计算机编程语言，它可以让你编写一次代码，然后在任何设备上运行（Write Once, Run Anywhere）。

#### 1.2.2 Java的基本语法

**变量**：就像我们生活中用盒子装东西一样，变量是用来存储数据的容器。
```java
int age = 18;       // 整数类型变量
String name = "张三";  // 字符串类型变量
boolean isStudent = true; // 布尔类型变量（true/false）
```

**类和对象**：类是对现实世界中事物的描述，对象是类的具体实例。
```java
// 类定义
class Student {
    String name;
    int age;
    
    void study() {
        System.out.println(name + "正在学习");
    }
}

// 创建对象
Student student = new Student();
student.name = "李四";
student.age = 20;
student.study();  // 调用方法
```

**方法**：方法是一段执行特定任务的代码块。
```java
// 定义方法
int add(int a, int b) {
    return a + b;
}

// 调用方法
int result = add(5, 3);  // result = 8
```

#### 1.2.3 控制流程

**条件语句**：根据条件决定执行哪段代码。
```java
if (age >= 18) {
    System.out.println("已成年");
} else {
    System.out.println("未成年");
}
```

**循环语句**：重复执行一段代码。
```java
// for循环
for (int i = 0; i < 5; i++) {
    System.out.println(i);
}

// while循环
int count = 0;
while (count < 5) {
    System.out.println(count);
    count++;
}
```

## 2. Spring Boot框架基础

### 2.1 什么是Spring Boot？
Spring Boot是一个基于Spring的框架，它可以帮助我们快速创建独立的、生产级别的Spring应用程序。简单来说，它让我们更容易地开发Java Web应用。

### 2.2 Spring Boot的核心特性

- **自动配置**：根据你添加的依赖自动配置Spring应用
- **起步依赖**：提供常用依赖的集合，简化配置
- **内嵌服务器**：内置Tomcat、Jetty等Web服务器
- **无代码生成和XML配置**：减少繁琐的配置工作

### 2.3 Spring Boot项目结构

一个典型的Spring Boot项目结构如下：
```
src/
  ├── main/
  │   ├── java/com/example/taskmanagement/
  │   │   ├── controller/   # 控制器层，处理HTTP请求
  │   │   ├── service/      # 服务层，业务逻辑
  │   │   ├── mapper/       # 数据访问层
  │   │   ├── model/        # 数据模型
  │   │   ├── config/       # 配置类
  │   │   └── TaskManagementApiApplication.java  # 启动类
  │   └── resources/        # 资源文件
  │       ├── mappers/      # MyBatis映射文件
  │       ├── application.yml  # 应用配置
  │       └── schema.sql    # 数据库表结构
  └── test/                 # 测试代码
```

### 2.4 Spring Boot注解

- `@SpringBootApplication`：标记主应用类，包含自动配置、组件扫描等
- `@RestController`：标记一个类为REST API控制器
- `@RequestMapping`：映射HTTP请求到控制器方法
- `@Autowired`：自动注入依赖
- `@Service`：标记服务层组件

### 2.5 REST API基础

REST（Representational State Transfer）是一种软件架构风格，用于构建Web服务。REST API通过HTTP协议提供资源的访问。

常用的HTTP方法：
- `GET`：获取资源
- `POST`：创建新资源
- `PUT`：更新资源
- `DELETE`：删除资源

## 3. MyBatis框架基础

### 3.1 什么是MyBatis？
MyBatis是一个Java持久层框架，它可以帮助我们更方便地操作数据库。它支持自定义SQL、存储过程以及高级映射。

### 3.2 MyBatis的核心组件

- **SqlSessionFactory**：创建SqlSession的工厂
- **SqlSession**：执行SQL命令的会话
- **Mapper接口**：定义数据库操作的接口
- **XML映射文件**：包含SQL语句和结果映射

### 3.3 MyBatis的使用步骤

1. 定义实体类（Model）
2. 创建Mapper接口
3. 编写XML映射文件
4. 配置MyBatis
5. 注入Mapper并使用

### 3.4 动态SQL

MyBatis的动态SQL允许我们根据条件动态构建SQL语句，非常灵活。

常用的动态SQL标签：
- `<if>`：条件判断
- `<where>`：动态WHERE子句
- `<choose>/<when>/<otherwise>`：相当于switch-case
- `<foreach>`：循环遍历集合

## 4. MySQL数据库基础

### 4.1 什么是数据库？
数据库是一个有组织的数据集合。就像我们的笔记本，但是更高效、更结构化地存储和管理数据。

### 4.2 MySQL基本概念

- **数据库**：存储相关数据的容器
- **表**：数据库中的数据集合，由行和列组成
- **行**：表中的一条记录
- **列**：表中的一个字段
- **主键**：唯一标识表中每条记录的字段

### 4.3 MySQL基本操作

**创建数据库**：
```sql
CREATE DATABASE taskdb;
```

**使用数据库**：
```sql
USE taskdb;
```

**创建表**：
```sql
CREATE TABLE tasks (
    id INT AUTO_INCREMENT PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    description TEXT,
    completed BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);
```

**插入数据**：
```sql
INSERT INTO tasks (title, description) VALUES ('学习Java', '每天学习2小时');
```

**查询数据**：
```sql
SELECT * FROM tasks;
SELECT * FROM tasks WHERE completed = false;
```

**更新数据**：
```sql
UPDATE tasks SET completed = true WHERE id = 1;
```

**删除数据**：
```sql
DELETE FROM tasks WHERE id = 1;
```

## 5. 单元测试基础

### 5.1 什么是单元测试？
单元测试是对软件中最小可测试单元进行检查和验证的测试。在Java中，通常是测试一个类或方法。

### 5.2 JUnit 5基础

JUnit 5是Java中最流行的单元测试框架。

**基本注解**：
- `@Test`：标记一个方法为测试方法
- `@BeforeEach`：在每个测试方法执行前运行
- `@AfterEach`：在每个测试方法执行后运行
- `@BeforeAll`：在所有测试方法执行前运行一次
- `@AfterAll`：在所有测试方法执行后运行一次

**断言方法**：
- `assertEquals(expected, actual)`：验证两个值是否相等
- `assertTrue(condition)`：验证条件是否为真
- `assertFalse(condition)`：验证条件是否为假
- `assertNull(object)`：验证对象是否为null
- `assertNotNull(object)`：验证对象是否不为null

### 5.3 Mockito框架

Mockito是一个用于Java的模拟框架，可以创建和配置模拟对象。

**基本用法**：
- `@Mock`：创建一个模拟对象
- `@InjectMocks`：将模拟对象注入到测试对象中
- `when().thenReturn()`：配置模拟对象的行为
- `verify()`：验证方法是否被调用

## 7. Task管理系统架构解析

### 7.1 系统架构图

```
客户端(浏览器/Postman) → Controller层 → Service层 → Mapper层 → 数据库
```

### 7.2 各层职责

- **Controller层**：接收HTTP请求，返回HTTP响应
- **Service层**：处理业务逻辑
- **Mapper层**：与数据库交互
- **Model层**：定义数据模型

### 7.3 核心功能说明

- **任务列表查询**：支持分页、搜索和过滤
- **任务详情查询**：根据ID获取任务详情
- **创建任务**：添加新任务
- **更新任务**：修改任务信息
- **删除任务**：删除单个或多个任务
- **更新任务状态**：标记任务为已完成或未完成
